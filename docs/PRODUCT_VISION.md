# Blink Product Vision

## Product Mission

To build the absolute fastest, most lightweight Android document viewer for everyday users. Blink is designed to handle opening standard documents instantly, providing a seamless and focus-oriented reading experience without overhead.

## Product Philosophy

**"Tap → Open → Read"**

Every design decision, library integration, and line of code must align with this philosophy. 
* **Zero friction**: No sign-ups, no splash screens, no internet checks, no upsell prompts.
* **Extreme Speed**: Minimize time-to-first-render. Tap a document in any app, and see the content immediately.
* **Low Footprint**: Be respectful of device storage and RAM. Keep APK size minimal and runtime memory allocations low.
* **Native Integration**: Function as a transparent, high-performance extension of the Android system.

## User Goals

* **Instant Attachment View**: Instantly open PDF, Office, and text document attachments received via WhatsApp, Telegram, Gmail, Slack, and other apps.
* **Fast Local Document Access**: View files downloaded to local directories (e.g., the Downloads folder) without waiting.
* **Optimized Reading Experience**: Scroll smoothly through large files (e.g., long PDFs or massive sheets) without lag or stutter.
* **Offline Operations**: View any local document completely offline with total privacy and security.

## Non-Goals

Blink is explicitly NOT:
1. **An Office Suite**: No editing, creation, formatting, or manipulation of documents.
2. **A PDF Editor**: No signing, annotations, page reordering, highlighting, or form filling.
3. **A Cloud Storage Platform**: No syncing, cloud accounts, remote directories, or backend storage integrations.
4. **An AI Tool**: No summarization, OCR, transcription, translation, or semantic search.
5. **An Ad/Subscription Platform**: No paywalls, user tracking, or advertising SDKs.
